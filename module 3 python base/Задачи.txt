Задача 1 — Простейший геттер

Цель: познакомиться с @property.

Класс Person с приватным атрибутом _age.

Сделать геттер age, который возвращает _age.

Создать объект и вывести возраст через геттер.

Подсказка: используем @property.

***************************************************************
Задача 2 — Геттер с логированием

Цель: показать, что геттер может что-то делать при чтении.

Класс BankAccount с приватным _balance.

Геттер balance выводит сообщение "Баланс запрошен" перед возвратом _balance.

Создать объект и проверить вывод.

****************************************************************
Задача 3 — Сеттер с простой проверкой

Цель: контролировать запись значения.

Класс Person с _age.

Геттер age.

Сеттер age проверяет: возраст не может быть меньше 0.

Если передано отрицательное число, выбросить ValueError.

************************************************************************
Задача 4 — Сеттер с проверкой типа

Цель: комбинировать проверку значений и типов.

Класс Temperature с _celsius.

Геттер celsius.

Сеттер celsius проверяет:

Значение должно быть числом (int или float)

Температура не ниже абсолютного нуля (-273.15)

Если проверка не проходит, выбросить подходящее исключение.

***********************************************************************
Задача 5 — Вычисляемое свойство

Цель: научиться создавать свойства, которые вычисляются на лету.

Класс Rectangle с приватными _width и _height.

Геттеры и сеттеры для ширины и высоты.

Свойство area (только геттер), которое возвращает площадь: width * height.

Пользователь не должен напрямую задавать area.

*************************************************************************
Задача 6 — Зависимое свойство

Цель: показываем, что сеттер одного свойства может влиять на другое.

Класс Rectangle с _width и _height.

Свойство width и height с геттерами/сеттерами.

Свойство perimeter (только геттер), которое вычисляет периметр: 2*(width + height).

Изменение ширины/высоты автоматически меняет периметр (без отдельного сеттера для периметра).

*****************************************************************************************
Задача 7 — Защищённая логика с внутренним кэшированием

Цель: более сложная логика внутри геттера/сеттера.

Класс Employee с _salary и _bonus.

Свойство salary:

Геттер возвращает _salary + _bonus.

Сеттер проверяет, что _salary ≥ 0.

Свойство bonus:

Геттер возвращает _bonus.

Сеттер проверяет, что _bonus ≥ 0 и не больше 50% от _salary.

При попытке установить неправильное значение выбрасывается исключение.

💡 Эти задачи идут по нарастающей сложности:

Просто геттер

Геттер с эффектом (лог)

Сеттер с проверкой

Сеттер с типом и диапазоном

Вычисляемое свойство

Свойство, зависящее от других атрибутов

Сложная логика + внутренние проверки
